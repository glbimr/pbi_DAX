Section A

Import the given files 'Sales' and 'Product' in Power BI. Give all answers from Q.1(A) to Q.1(F) on one page named Q.1.

12 Marks

Q. 1: (A) Text Box & Total Sales by Region Chart

Bloom: L3

Goal: Add a text box stating "Answer 1" and create a formatted chart showing total sales by region.

  1. Go to Insert > Text box from the top ribbon. Type "Answer 1" and increase the font size.
  2. From the Visualizations pane, select Clustered Column Chart (or Donut Chart).
  3. Drag Region (from Sales table) into the X-axis (or Legend).
  4. Drag Sales (from Sales table) into the Y-axis (or Values).
  5. Formatting: Go to the "Format your visual" tab (paint roller/brush icon). Turn on Data Labels, add a distinct Title, and round the corners in General > Effects > Visual border.

Q. 1: (B) Combo Chart & Slicer

Bloom: L3

Goal: Create a combo chart for sales and profit by customers, and use a slicer for region.

  1. Select the Line and Stacked Column Chart icon from the Visualizations pane.
  2. Drag Customer to the X-axis.
  3. Drag Sales to the Column y-axis.
  4. Drag Profit to the Line y-axis.
  5. Click on a blank space on the canvas, then select the Slicer visual.
  6. Drag Region into the Slicer field. Change slicer settings to "Dropdown" or "Tile" for better formatting.

Q. 1: (C) DAX Measure for Row Count

Bloom: L3

Goal: Create a measure using DAX to count rows in 'Sales' and show it on a Card.

  1. Right-click the Sales table in the Data pane and select New Measure.
  2. Type the following DAX formula in the formula bar:
DAX
Q1(C) = COUNTROWS('Sales')
  1. Select the Card visual from the Visualizations pane.
  2. Drag your newly created measure Q1(C) into the Card's field.

Q. 1: (D) Area Chart for Monthly Trends

Bloom: L3

Goal: Area chart showing monthly sales trends with formatting.

  1. Select the Area Chart visual.
  2. Drag Date to the X-axis. (Ensure it uses the Date Hierarchy, remove Day and Quarter to just show Year and Month).
  3. Drag Sales to the Y-axis.
  4. Formatting: Go to Format visual > Lines > Colors. Change the fill opacity to make it look professional. Add markers if desired.

Q. 1: (E) Page Navigation Buttons

Bloom: L3

Goal: Create page navigation buttons for every question page.

  1. Go to Insert > Buttons > Navigator > Page Navigator.
  2. Power BI will automatically create a button for every page in your report.
  3. Go to Format > Shape to change the button shapes (e.g., Pill or Rounded Rectangle). Customize the Fill colors for Default, Hover, and Selected states.

Q. 1: (F) Waterfall Chart for Profit

Bloom: L3

Goal: Waterfall chart showing profit contribution by product.

  1. Select the Waterfall Chart visual.
  2. Drag ProductID (or ProductName if merged) to the Category field.
  3. Drag Profit to the Y-axis.
  4. Formatting: Turn on Data Labels. Go to Format > Columns and set distinct colors for Increases (Green) and Decreases (Red).

Question 4 (Choose A or B)

Q. 4: (A) DAX: LOOKUPVALUE

Bloom: L4

Goal: Fetch product category names into 'Sales'.

  1. Go to Data View. Select Sales table.
  2. Click New Column. Enter DAX:
DAX
Q4(A) = LOOKUPVALUE('Product'[Category], 'Product'[ProductID], 'Sales'[ProductID])
OR

Q. 4: (B) DAX: IF Statement

Bloom: L4

Goal: Label sales as 'High' if above 1000, else 'Low'.

  1. Go to Data View. Select Sales table.
  2. Click New Column. Enter DAX:
DAX
Q4(B) = IF(Sales[Sales] > 1000, "High", "Low")

Section C

Case Study: Hospital Patient Data Dashboard. The whole question 5 must be on one page named 'Q.5'.

10 Marks

Case Context

You are a data analyst at a multi-specialty hospital. Management wants to monitor patient treatment patterns and hospital performance. You must create an insightful and interactive Power BI dashboard on a single page.

Q. 5: (A) DAX & Core Dashboard Cards

Bloom: L5

Goal: Create cards for Total Patients, Average Treatment Cost, and Average Length of Stay. Add slicers.

Step 1: Calculate Length of Stay

First, you need a new column or measure to calculate the days between Admission and Discharge.

Create a New Column in the Hospital table:

DAX
Length of Stay = DATEDIFF('Hospital_Patient_Data'[Admission Date], 'Hospital_Patient_Data'[Discharge Date], DAY)
  1. Card 1 (Total Patients): Select Card visual. Drag PatientID to fields. Change aggregation to Count (Distinct).
  2. Card 2 (Avg Treatment Cost): Select Card visual. Drag Treatment Cost. Change aggregation to Average. Format as Currency.
  3. Card 3 (Avg Length of Stay): Select Card visual. Drag the new Length of Stay column. Change aggregation to Average.
  4. Slicers: Add 2-3 slicers at the top or side of your page. Good candidates are Diagnosis, Gender, or a Date Slicer using Admission Date.

Q. 5: (B) Insightful Visualizations

Bloom: L5

Goal: Create 5-6 visualizations to get the best insights out of data.

Arrange these neatly on the same page as 5(A) to form a cohesive dashboard. Ensure consistent colors and borders.

1. Patients by Diagnosis

Visual: Donut Chart or Pie Chart

Config: Legend: Diagnosis, Values: Count of PatientID.

2. Cost vs. Diagnosis

Visual: Clustered Bar Chart

Config: Y-axis: Diagnosis, X-axis: Average Treatment Cost.

3. Admissions Over Time

Visual: Line Chart

4. Patient Demographics (Age)

Visual: Column Chart

5. Gender Distribution
